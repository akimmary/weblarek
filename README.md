# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
#### Интерфейс IProduct — Товар
Назначение: описывает структуру товара, отображаемого в каталоге и добавляемого в корзину.

`interface IProduct` {
  `id: string;` - Уникальный идентификатор товара
  `description: string;` - Описание товара
  `image: string;` - Изображение
  `title: string;` - Название товара
  `category: string;` - Категория товара 
  `price: number | null;` - Цена товара (null означает "бесценно")
}

#### Интерфейс IBuyer — Покупатель
Назначение: хранит информацию о покупателе, необходимую для оформления и доставки заказа.

type TPayment = 'card' | 'cash' | ''; - Способ оплаты: карта, наличные или не выбран;

`interface IBuyer` {
  `payment: TPayment;` - Способ оплаты
  `email: string;` - Электронная почта
  `phone: string;` - Контактный телефон
  `address: string;` - Адрес доставки
}

### Модели данных

#### Класс `CatalogModel`

Назначение и зона ответственности: 
Хранение всех товаров, доступных в приложении, и управление выбранным товаром для подробного просмотра.

Конструктор:
`constructor()` — параметров не принимает.

Поля класса:
`_products: IProduct[]` — хранит массив всех товаров, доступных в каталоге
`_selectedProduct: IProduct | null` — хранит товар, выбранный для подробного отображения. `null` означает, что ни один товар не выбран

Методы класса:
`setProducts(products: IProduct[]): void` — сохраняет массив товаров, полученный в параметрах, в поле `_products`
`getProducts(): IProduct[]` — возвращает массив всех товаров из поля `_products`
`getProductById(id: string): IProduct | undefined` — возвращает товар с указанным `id` или `undefined`, если товар не найден
`setSelectedProduct(product: IProduct | null): void` — сохраняет товар для подробного отображения. При передаче `null` сбрасывает выбор
`getSelectedProduct(): IProduct | null` — возвращает выбранный товар или `null`

#### Класс `CartModel`

Назначение и зона ответственности: 
Хранение товаров, которые пользователь выбрал для покупки, и предоставление методов для управления содержимым корзины.

Конструктор: 
`constructor()` — параметров не принимает. При создании экземпляра инициализирует пустой массив товаров.

Поля класса:
`_items: IProduct[]` — хранит массив товаров, выбранных покупателем для покупки

Методы класса:
`getItems(): IProduct[]` — возвращает массив товаров, которые находятся в корзине
`addProduct(product: IProduct): void` — добавляет товар, полученный в параметре, в массив корзины
`removeProduct(product: IProduct): void` — удаляет товар, полученный в параметре, из массива корзины.
`clear(): void` — полностью очищает корзину, присваивая полю _items пустой массив
`getTotalPrice(): number` — вычисляет и возвращает сумму стоимостей всех товаров в корзине.
`getTotalCount(): number` — возвращает общее количество товаров в корзине (длину массива _items)
`containsProduct(id: string): boolean` — принимает идентификатор товара и проверяет, есть ли в корзине товар с таким id. Возвращает true, если товар присутствует, иначе false

#### Класс `BuyerModel`

Назначение и зона ответственности:
Хранение данных покупателя (вид оплаты, адрес, телефон, email), которые тот указывает при оформлении заказа. Обеспечивает валидацию данных.

Конструктор:
`constructor()` — параметров не принимает. При создании экземпляра инициализирует поле `_data` с пустыми значениями.

Поля класса:
`_data: IBuyer` — хранит объект с данными покупателя. Начальное значение: `{ payment: '', email: '', phone: '', address: '' }`

Методы класса:
`setPayment(payment: TPayment): void` — сохраняет способ оплаты в поле` _data.payment`
`setEmail(email: string): void` — сохраняет email в поле `_data.email`
`setPhone(phone: string): void` — сохраняет номер телефона в поле `_data.phone`
`setAddress(address: string): void` — сохраняет адрес доставки в поле `_data.address`
`getAllData(): IBuyer` — возвращает копию всех данных покупателя из поля `_data`
`clear(): void` — очищает все данные покупателя, сбрасывая каждое поле `_data` в пустое значение
`validate(): Partial<Record<keyof IBuyer, string>>` — проверяет валидность каждого поля. Поле считается валидным, если оно не пустое. Возвращает объект, где ключи — названия полей с ошибкой, а значения — текст ошибки. Если поле валидно, оно отсутствует в возвращаемом объекте

Правила валидации метода `validate()`: 
`payment` — не пустая строка. Ошибка: 'Не выбран вид оплаты'
`email` — не пустая строка. Ошибка: 'Укажите email'
`phone` — не пустая строка. Ошибка: 'Укажите телефон'
`address` — не пустая строка. Ошибка: 'Укажите адрес'

Пример возвращаемого значения:
{
  "payment": "Не выбран вид оплаты",
  "email": "Укажите email"
}

### Слой коммуникации

#### Класс `AppApi`

Назначение и зона ответственности:
Отвечает за взаимодействие с сервером API «веб-ларёк». Использует композицию с объектом, реализующим интерфейс `IApi`, для выполнения HTTP-запросов.

Конструктор:
`constructor(api: IApi)` — принимает объект, соответствующий интерфейсу `IApi`, который предоставляет методы `get` и `post`.

Поля класса:
`_api: Api` — хранит экземпляр API-клиента для выполнения запросов к серверу

Методы класса:
`getProducts(): Promise<IProduct[]>` — отправляет GET-запрос на эндпоинт `/product` для получения списка всех товаров. Возвращает промис с массивом товаров
`postOrder(orderData: TOrderData): Promise<TOrderResult>` — отправляет POST-запрос на эндпоинт `/order` с данными заказа. Возвращает промис с результатом оформления заказа

### Слой Представления 

#### Класс Header

Назначение и зона ответственности:
Отвечает за отображение шапки сайта: управление счётчиком товаров в корзине и кнопкой корзины.
Наследуется от: Component<IHeader>

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент шапки сайта и событие.

Поля класса:
`basketButton: HTMLButtonElement` — кнопка открытия корзины .header__basket.
`countElement: HTMLElement` — элемент для отображения количества товаров в корзине .header__basket-counter.

Методы класса:
`set counter(value: number)` — сеттер, устанавливает переданное значение в countElement (отображает количество товаров в корзине)

Генерируемые события:
`basket:open` — при клике на кнопку корзины

#### Класс Gallery

Назначение и зона ответственности:
Отвечает за отображение каталога товаров на главной странице. Принимает массив HTML-элементов карточек и отображает их в контейнере.
Наследуется от: Component<IGallery>

Конструктор:
`constructor(container: HTMLElement)` — принимает DOM-элемент — контейнер для каталога товаров

Поля класса:
`catalogElement: HTMLElement` — элемент-контейнер, в который будут помещены карточки товаров .gallery

Методы класса:
`set catalog(items: HTMLElement[])` — сеттер, очищает catalogElement и добавляет в него все переданные элементы карточек

#### Класс Modal

Назначение и зона ответственности:
Управляет отображением модального окна. Позволяет динамически заменять содержимое.
Наследуется от: Component<IModal>

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент модального окна и событие

Поля класса:
`modalButton: HTMLButtonElement` — кнопка закрытия модального окна .modal__close
`contentElement: HTMLElement` — контейнер для динамического содержимого .modal__content.

Методы класса:
`set content(value: HTMLElement): void` — заменяет содержимое модального окна на переданный элемент
`open(): void` — открывает модальное окно (добавляет класс `modal_active`)
`close(): void` — закрывает модальное окно (удаляет класс `modal_active`) и очищает содержимое

Генерируемые события:
`modal:close` — при клике на кнопку закрытия модального окна

#### Класс OrderSuccess

Назначение и зона ответственности:
Отвечает за отображение модального окна с подтверждением успешного оформления заказа. Отображает сумму списанных синапсов и кнопку закрытия.
Наследуется от: Component<ISuccess>

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент корневого контейнера модального окна успешного заказа и событие

Поля класса:
`closeButton: HTMLButtonElement` — кнопка закрытия с текстом «За новыми покупками!»
`descriptionElement: HTMLElement` — элемент с описанием списанных синапсов 

Методы класса:
`set total(value: number): void` — сеттер, устанавливает переданную сумму в текст описания: Списано {value} синапсов

Генерируемые события:
`success:close` — при клике на кнопку «За новыми покупками!»

#### Базовый класс Card<T>

Назначение и зона ответственности:
Базовый класс для всех типов карточек товара. Содержит общие свойства и методы, которые переиспользуются в карточке каталога, карточке превью (модальное окно) и карточке корзины.
Наследуется от: Component<IProduct>

Конструктор:
`constructor(container: HTMLElement)` — принимает DOM-элемент карточки

Поля класса:
`titleElement: HTMLElement` — элемент с названием товара 
`priceElement: HTMLElement` — элемент с ценой товара
`categoryElement?: HTMLElement` — элемент с категорией товара.
`imageElement?: HTMLImageElement` — элемент с изображением товара. 

Методы класса:
`set title(value: string): void` — устанавливает название товара
`set price(value: number | null): void` — устанавливает цену. Если value === null, отображается текст «Бесценно»
`set image(value: string): void` — устанавливает изображение 
`set category(value: string): void` — меняет категорию, добавляет новый CSS-класс для правильного цвета фона

Методы класса:

Генерируемые события:

`set inCart(value: boolean): void` — меняет состояние кнопки
`set data(value: ICardPreview): void` — заполняет карточку всеми данными

Генерируемые события:

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент карточки и обработчик клика на кнопку удаления

`indexElement: HTMLElement` — порядковый номер

`set index(value: number): void` — устанавливает порядковый номер
`set data(value: ICardBasket): void` — заполняет карточку данными (название, цена, индекс)

Генерируемые события:
`card:delete`	при клике на кнопку удаления (передаёт id товара)

#### Класс Basket

Назначение и зона ответственности:
Отображает содержимое корзины: список добавленных товаров, общую сумму и кнопку оформления заказа.
Наследуется от: Component<IBasket>

Конструктор: 
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент корзины и опциональный колбэк для кнопки «Оформить»

Поля класса:
`listElement: HTMLElement` — элемент, контейнер для списка карточек товаров
`priceElement: HTMLElement` — элемент, отображает общую сумму
`orderButton: HTMLButtonElement` — кнопка для оформления заказа

Методы класса:
`set items(items: HTMLElement[]): void` — очищает listElement и добавляет в него переданные элементы карточек
`set total(value: number): void` — устанавливает текст в priceElement в формате {value} синапсов
`set disabled(value: boolean): void` — блокирует/разблокирует кнопку orderButton

#### Базовый класс Form<T>

Назначение и зона ответственности:
Базовый класс для всех форм. Управляет полями ввода, валидацией, состоянием кнопки отправки и отображением ошибок.
Наследуется от: Component<T>

Конструктор:
`constructor(container: HTMLFormElement, events: IEvents)`
- container: форма (HTMLFormElement)
- events: брокер событий для генерации событий при изменении полей и отправке формы

Поля класса:
`submitButton: HTMLButtonElement` — кнопка отправки 
`errorElement: HTMLElement` — элемент для вывода ошибок 

Методы класса:
`protected setInputValue(name: keyof T, value: string): void` — устанавливает значение поля по имени
`set valid(value: boolean): void` — блокирует/разблокирует кнопку отправки
`set errors(value: string): void` — устанавливает текст ошибок

### Слой презентера

Назначение и зона ответственности:
Отвечает за связь между моделями данных и представлениями. Обрабатывает события, 
генерируемые моделями и представлениями, и обновляет интерфейс.

Реализация:
Презентер реализован в основном скрипте приложения `main.ts`, так как приложение 
имеет одну страницу и не требует усложнения архитектуры.

Обрабатываемые события:

| Событие | Источник | Действие |
|---------|----------|----------|
| `catalog:changed` | `CatalogModel` | Обновить галерею |
| `catalog:selected` | `CatalogModel` | Сохранить выбранный товар |
| `cart:changed` | `CartModel` | Обновить счётчик и корзину |
| `buyer:changed` | `BuyerModel` | (внутреннее) |
| `card:select` | `CardCatalog` | Открыть карточку товара |
| `card:add` | `CardPreview` | Добавить товар в корзину |
| `card:remove` | `CardPreview` | Удалить товар из корзины |
| `card:delete` | `CardBasket` | Удалить товар из корзины |
| `basket:open` | `Header` | Открыть корзину |
| `modal:close` | `Modal` | Закрыть модальное окно |
| `order:submit` | `Basket` / `OrderForm` | Открыть форму заказа / переход к контактам |
| `order:input` | `OrderForm` | Валидация формы заказа |
| `contacts:input` | `ContactsForm` | Валидация формы контактов |
| `contacts:submit` | `ContactsForm` | Отправить заказ |
| `success:close` | `OrderSuccess` | Закрыть окно успеха |

Основной поток работы:

1. Загрузка товаров с сервера → сохранение в `CatalogModel` → генерация `catalog:changed`
2. Отображение галереи товаров
3. Клик по карточке → генерация `card:select` → открытие модального окна с `CardPreview`
4. Клик «В корзину» → генерация `card:add` → добавление в `CartModel`
5. Клик по корзине в шапке → генерация `basket:open` → открытие модального окна с корзиной
6. Нажатие «Оформить» → генерация `order:submit` → открытие формы заказа
7. Заполнение формы заказа → `order:input` → валидация и сохранение данных
8. Нажатие «Далее» → генерация `order:submit` → переход к форме контактов
9. Заполнение формы контактов → `contacts:input` → валидация и сохранение данных
10. Нажатие «Оплатить» → генерация `contacts:submit` → отправка заказа на сервер, очистка корзины, открытие окна успеха
11. Нажатие «За новыми покупками» → генерация `success:close` → закрытие модального окна и очистка корзины
