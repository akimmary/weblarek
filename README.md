# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

 ### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
#### Интерфейс IProduct — Товар
Назначение: описывает структуру товара, отображаемого в каталоге и добавляемого в корзину.

`interface IProduct` {
  `id: string;` - Уникальный идентификатор товара
  `description: string;` - Описание товара
  `image: string;` - Изображение
  `title: string;` - Название товара
  `category: string;` - Категория товара 
  `price: number | null;` - Цена товара (null означает "бесценно")
}

#### Интерфейс IBuyer — Покупатель
Назначение: хранит информацию о покупателе, необходимую для оформления и доставки заказа.

type TPayment = 'card' | 'cash' | ''; - Способ оплаты: карта, наличные или не выбран;

`interface IBuyer` {
  `payment: TPayment;` - Способ оплаты
  `email: string;` - Электронная почта
  `phone: string;` - Контактный телефон
  `address: string;` - Адрес доставки
}

### Модели данных

#### Класс `CatalogModel`

Назначение и зона ответственности: 
Хранение всех товаров, доступных в приложении, и управление выбранным товаром для подробного просмотра.

Конструктор:
`constructor(events: IEvents)` — принимает брокер событий для генерации событий при изменении каталога.

Поля класса:
`_products: IProduct[]` — хранит массив всех товаров, доступных в каталоге
`_selectedProduct: IProduct | null` — хранит товар, выбранный для подробного отображения. `null` означает, что ни один товар не выбран
`events: IEvents` — брокер событий

Методы класса:
`setProducts(products: IProduct[]): void` — сохраняет массив товаров, полученный в параметрах, в поле `_products`
`getProducts(): IProduct[]` — возвращает копию массива всех товаров из каталога
`getProductById(id: string): IProduct | undefined` — возвращает товар с указанным `id` или `undefined`, если товар не найден
`setSelectedProduct(product: IProduct | null): void` — сохраняет товар для подробного отображения. При передаче `null` сбрасывает выбор
`getSelectedProduct(): IProduct | null` — возвращает выбранный товар или `null`

#### Класс `CartModel`

Назначение и зона ответственности: 
Хранение товаров, которые пользователь выбрал для покупки, и предоставление методов для управления содержимым корзины.

Конструктор: 
`constructor(events: IEvents)` — принимает брокер событий для генерации событий при изменении корзины.

Поля класса:
`_items: IProduct[]` — хранит массив товаров, выбранных покупателем для покупки
`events: IEvents` — брокер событий

Методы класса:
`getItems(): IProduct[]` — возвращает копию массива товаров, которые находятся в корзине
`addProduct(product: IProduct): void` — добавляет товар, полученный в параметре, в массив корзины
`removeProduct(product: IProduct): void` — удаляет товар, полученный в параметре, из массива корзины.
`clear(): void` — полностью очищает корзину, присваивая полю _items пустой массив
`getTotalPrice(): number` — вычисляет и возвращает сумму стоимостей всех товаров в корзине.
`getTotalCount(): number` — возвращает общее количество товаров в корзине (длину массива _items)
`containsProduct(id: string): boolean` — принимает идентификатор товара и проверяет, есть ли в корзине товар с таким id. Возвращает true, если товар присутствует, иначе false

Генерируемые события:
`cart:changed` — при любом изменении содержимого корзины (добавление, удаление, очистка)

#### Класс `BuyerModel`

Назначение и зона ответственности:
Хранение данных покупателя (вид оплаты, адрес, телефон, email), которые тот указывает при оформлении заказа. Обеспечивает валидацию данных.

Конструктор:
`constructor(events: IEvents)` — принимает брокер событий для генерации событий при изменении данных.

Поля класса:
`_data: IBuyer` — хранит объект с данными покупателя. Начальное значение: `{ payment: '', email: '', phone: '', address: '' }`
`events: IEvents` — брокер событий

Методы класса:
`setPayment(payment: TPayment): void` — сохраняет способ оплаты. Генерирует событие `buyer:changed`
`setEmail(email: string): void` — сохраняет email. Генерирует событие `buyer:changed`
`setPhone(phone: string): void` — сохраняет номер телефона. Генерирует событие `buyer:changed`
`setAddress(address: string): void` — сохраняет адрес доставки. Генерирует событие `buyer:changed`
`getAllData(): IBuyer` — возвращает копию всех данных покупателя
`clear(): void` — очищает все данные покупателя. Генерирует событие `buyer:changed`
`validate(): TValidationErrors` — проверяет валидность каждого поля. Поле считается валидным, если оно не пустое. Возвращает объект с ошибками

Генерируемые события:
`buyer:changed` — при любом изменении данных покупателя

Правила валидации метода `validate()`: 
`payment` — не пустая строка. Ошибка: 'Не выбран вид оплаты'
`email` — не пустая строка. Ошибка: 'Укажите email'
`phone` — не пустая строка. Ошибка: 'Укажите телефон'
`address` — не пустая строка. Ошибка: 'Укажите адрес'

Пример возвращаемого значения:
{
  "payment": "Не выбран вид оплаты",
  "email": "Укажите email"
}

### Слой коммуникации

#### Класс `AppApi`

Назначение и зона ответственности:
Отвечает за взаимодействие с сервером API «веб-ларёк». Использует композицию с объектом, реализующим интерфейс `IApi`, для выполнения HTTP-запросов.

Конструктор:
`constructor(api: IApi)` — принимает объект, соответствующий интерфейсу `IApi`, который предоставляет методы `get` и `post`.

Поля класса:
`_api: Api` — хранит экземпляр API-клиента для выполнения запросов к серверу

Методы класса:
`getProducts(): Promise<IProduct[]>` — отправляет GET-запрос на эндпоинт `/product` для получения списка всех товаров. Возвращает промис с массивом товаров
`postOrder(orderData: TOrderData): Promise<TOrderResult>` — отправляет POST-запрос на эндпоинт `/order` с данными заказа. Возвращает промис с результатом оформления заказа

### Слой Представления 

#### Класс Header

Назначение и зона ответственности:
Отвечает за отображение шапки сайта: управление счётчиком товаров в корзине и кнопкой корзины.
Наследуется от: Component<IHeader>

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент шапки сайта и событие.

Поля класса:
`basketButton: HTMLButtonElement` — кнопка открытия корзины .header__basket.
`countElement: HTMLElement` — элемент для отображения количества товаров в корзине .header__basket-counter.

Методы класса:
`set counter(value: number)` — сеттер, устанавливает переданное значение в countElement (отображает количество товаров в корзине)

Генерируемые события:
`basket:open` — при клике на кнопку корзины

#### Класс Gallery

Назначение и зона ответственности:
Отвечает за отображение каталога товаров на главной странице. Принимает массив HTML-элементов карточек и отображает их в контейнере.
Наследуется от: Component<IGallery>

Конструктор:
`constructor(container: HTMLElement)` — принимает DOM-элемент — контейнер для каталога товаров

Поля класса:
`catalogElement: HTMLElement` — элемент-контейнер, в который будут помещены карточки товаров .gallery

Методы класса:
`set catalog(items: HTMLElement[])` — сеттер, очищает catalogElement и добавляет в него все переданные элементы карточек

#### Класс Modal

Назначение и зона ответственности:
Управляет отображением модального окна. Позволяет динамически заменять содержимое.
Наследуется от: Component<IModal>

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент модального окна и событие

Поля класса:
`modalButton: HTMLButtonElement` — кнопка закрытия модального окна .modal__close
`contentElement: HTMLElement` — контейнер для динамического содержимого .modal__content.

Методы класса:
`set content(value: HTMLElement): void` — заменяет содержимое модального окна на переданный элемент
`open(): void` — открывает модальное окно (добавляет класс `modal_active`)
`close(): void` — закрывает модальное окно (удаляет класс `modal_active`) и очищает содержимое

Примечание: События не генерируются. Управление открытием/закрытием происходит через прямые вызовы методов `open()` и `close()` из презентера.


#### Класс OrderSuccess

Назначение и зона ответственности:
Отвечает за отображение модального окна с подтверждением успешного оформления заказа. Отображает сумму списанных синапсов и кнопку закрытия.
Наследуется от: Component<ISuccess>

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент корневого контейнера модального окна успешного заказа и событие

Поля класса:
`closeButton: HTMLButtonElement` — кнопка закрытия с текстом «За новыми покупками!»
`descriptionElement: HTMLElement` — элемент с описанием списанных синапсов 

Методы класса:
`set total(value: number): void` — сеттер, устанавливает переданную сумму в текст описания: Списано {value} синапсов

Генерируемые события:
`success:close` — при клике на кнопку «За новыми покупками!»

#### Базовый класс Card<T>

Назначение и зона ответственности:
Базовый класс для всех типов карточек товара. Содержит общие свойства и методы, которые переиспользуются в карточке каталога, карточке превью (модальное окно) и карточке корзины.
Наследуется от: `Component<T>` (где `T extends IProduct`)

Конструктор:
`constructor(container: HTMLElement)` — принимает DOM-элемент карточки

Поля класса:
`titleElement: HTMLElement` — элемент с названием товара (`.card__title`)
`priceElement: HTMLElement` — элемент с ценой товара (`.card__price`)

Методы класса:
`set title(value: string): void` — устанавливает название товара
`set price(value: number | null): void` — устанавливает цену. Если `value === null`, отображается текст «Бесценно»


#### Класс CardCatalog (карточка каталога)

Назначение и зона ответственности:
Отображает товар в галерее каталога на главной странице. При клике на карточку генерирует событие `card:select`.
Наследуется от: `Card<IProduct>`

Конструктор:
`constructor(container: HTMLElement, events: IEvents, onClick: () => void)` — принимает DOM-элемент карточки, брокер событий и колбэк для обработки клика

Поля класса:
Наследует поля `Card` (`titleElement`, `priceElement`)
`categoryElement: HTMLElement` — элемент для отображения категории (`.card__category`)
`imageElement: HTMLImageElement` — элемент для отображения изображения (`.card__image`)

Методы класса:
`set category(value: string): void` — устанавливает категорию и применяет CSS-класс для фона
`set image(value: string): void` — устанавливает изображение (формирует полный путь через `CDN_URL`)

Обновление карточки:
Карточка обновляется через метод `render`, который принимает объект с полями `title`, `price`, `category`, `image`. `Component.render` автоматически вызывает соответствующие сеттеры через `Object.assign`.

Колбэк:
`onClick` — вызывается при клике на карточку. Презентер генерирует событие `card:select`

#### Класс CardPreview (карточка превью)

Назначение и зона ответственности:
Отображает детальную информацию о товаре в модальном окне. Включает категорию, изображение, описание и кнопку действия.
Наследуется от: `Card<ICardPreview>`

Конструктор:
`constructor(container: HTMLElement, events: IEvents, onAction: () => void)` — принимает DOM-элемент карточки, брокер событий и колбэк для действия (добавление/удаление из корзины)

Поля класса:
Наследует поля `Card` (`titleElement`, `priceElement`)
`categoryElement: HTMLElement` — элемент для отображения категории (`.card__category`)
`imageElement: HTMLImageElement` — элемент для отображения изображения (`.card__image`)
`descriptionElement: HTMLElement` — описание товара (`.card__text`)
`addButton: HTMLButtonElement` — кнопка действия (`.card__button`)

Методы класса:
`set category(value: string): void` — устанавливает категорию и применяет CSS-класс для фона
`set image(value: string): void` — устанавливает изображение (формирует полный путь через `CDN_URL`)
`set description(value: string): void` — устанавливает описание товара
`set buttonText(value: string): void` — устанавливает текст на кнопке
`set buttonDisabled(value: boolean): void` — блокирует/разблокирует кнопку

Обновление карточки:
Карточка обновляется через метод `render`, который принимает объект с полями `title`, `price`, `category`, `image`, `description`, `buttonText`, `buttonDisabled`. `Component.render` автоматически вызывает соответствующие сеттеры через `Object.assign`.

Колбэк:
`onAction` — вызывается при клике на кнопку. Презентер сам определяет, добавить или удалить товар из корзины

#### Класс CardBasket (карточка корзины)

Назначение и зона ответственности:
Отображает товар в списке корзины. Содержит порядковый номер, название, цену и кнопку удаления. Нет изображения и категории.

Наследуется от: `Card<ICardBasket>` (категория и изображение не используются)

Конструктор:
`constructor(container: HTMLElement, events: IEvents, onDelete: () => void)` — принимает DOM-элемент карточки, брокер событий и колбэк для удаления товара

Поля класса:
Наследует поля `Card` (`titleElement`, `priceElement`)
`indexElement: HTMLElement` — элемент для отображения порядкового номера (`.basket__item-index`)
`deleteButton: HTMLButtonElement` — кнопка удаления (`.basket__item-delete`)

Методы класса:
`set index(value: number): void` — устанавливает порядковый номер

Обновление карточки:
Карточка обновляется через метод `render`, который принимает объект с полями `title`, `price`, `index`. `Component.render` автоматически вызывает соответствующие сеттеры через `Object.assign`.

Колбэк:
`onDelete` — вызывается при клике на кнопку удаления. Презентер удаляет товар из корзины

#### Класс Basket

Назначение и зона ответственности:
Отображает содержимое корзины: список добавленных товаров, общую сумму и кнопку оформления заказа.
Наследуется от: `Component<IBasket>`

Конструктор: 
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент корзины и брокер событий

Поля класса:
`listElement: HTMLElement` — элемент-контейнер для списка карточек товаров (`.basket__list`)
`priceElement: HTMLElement` — элемент для отображения общей суммы (`.basket__price`)
`orderButton: HTMLButtonElement` — кнопка для оформления заказа (`.basket__button`)

Методы класса:
`set items(items: HTMLElement[]): void` — заменяет содержимое списка карточек (использует `replaceChildren`)
`set total(value: number): void` — устанавливает текст в `priceElement` в формате `{value} синапсов`
`set disabled(value: boolean): void` — блокирует/разблокирует кнопку `orderButton`

Генерируемые события:
`basket:checkout` — при клике на кнопку «Оформить»

#### Базовый класс Form<T>

Назначение и зона ответственности:
Базовый класс для всех форм. Управляет кнопкой отправки и отображением ошибок.
Наследуется от: `Component<T>`

Конструктор:
`constructor(container: HTMLFormElement, events: IEvents)`
`container`: форма (HTMLFormElement)
`events`: брокер событий (не используется в базовом классе, передаётся для дочерних)

Поля класса:**
`submitButton: HTMLButtonElement` — кнопка отправки (`button[type="submit"]`)
`errorElement: HTMLElement` — элемент для вывода ошибок (`.form__errors`)

Методы класса:
`set valid(value: boolean): void` — блокирует/разблокирует кнопку отправки
`set errors(value: string): void` — устанавливает текст ошибок

Примечание: События `input` и `submit` генерируются в дочерних классах (`OrderForm`, `ContactsForm`), так как у каждой формы свои данные.

Генерируемые события:

### Слой презентера

Назначение и зона ответственности:
Отвечает за связь между моделями данных и представлениями. Обрабатывает события, 
генерируемые моделями и представлениями, и обновляет интерфейс.

Реализация:
Презентер реализован в основном скрипте приложения `main.ts`, так как приложение 
имеет одну страницу и не требует усложнения архитектуры.

Обрабатываемые события:

| Событие | Источник | Действие |
|---------|----------|----------|
| `catalog:changed` | `CatalogModel` | Обновить галерею |
| `catalog:selected` | `CatalogModel` | Открыть карточку превью |
| `cart:changed` | `CartModel` | Обновить счётчик в шапке и содержимое корзины |
| `buyer:changed` | `BuyerModel` | Обновить валидацию и состояние кнопок форм |
| `card:select` | `CardCatalog` | Сохранить выбранный товар в модель |
| `basket:open` | `Header` | Открыть корзину |
| `basket:checkout` | `Basket` | Открыть форму заказа |
| `basket:item-delete` | `CardBasket` | Удалить товар из корзины |
| `preview:action` | `CardPreview` | Добавить или удалить товар из корзины |
| `order:submit` | `OrderForm` | Перейти к форме контактов |
| `order:input` | `OrderForm` | Сохранить данные формы заказа в модель |
| `contacts:input` | `ContactsForm` | Сохранить данные формы контактов в модель |
| `contacts:submit` | `ContactsForm` | Отправить заказ на сервер |
| `success:close` | `OrderSuccess` | Закрыть окно успеха |

Основной поток работы:

1. Загрузка товаров с сервера → `catalog.setProducts()` → `catalog:changed` → отображение галереи
2. Клик по карточке → `card:select` → `catalog.setSelectedProduct()` → `catalog:selected` → открытие `CardPreview`
3. Клик по кнопке в `CardPreview` → `preview:action` → проверка наличия товара в корзине → добавление или удаление → закрытие модального окна
4. Клик по корзине в шапке → `basket:open` → открытие модального окна с корзиной
5. Нажатие «Оформить» → `basket:checkout` → открытие формы заказа с актуальными данными из модели
6. Заполнение формы заказа → `order:input` → сохранение в `BuyerModel` → `buyer:changed` → валидация и обновление кнопки
7. Нажатие «Далее» → `order:submit` → открытие формы контактов с актуальными данными из модели
8. Заполнение формы контактов → `contacts:input` → сохранение в `BuyerModel` → `buyer:changed` → валидация и обновление кнопки
9. Нажатие «Оплатить» → `contacts:submit` → отправка заказа на сервер → очистка корзины и данных покупателя → открытие окна успеха
10. Нажатие «За новыми покупками» → `success:close` → закрытие модального окна
